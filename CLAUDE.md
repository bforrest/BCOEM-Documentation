## Commands

```bash
npm run dev          # dev server at localhost:4321
npm run build        # production build to ./dist
npm run preview      # preview built site
npm test             # run Vitest unit tests
npm run fetch-issues # fetch GitHub issues → src/data/issues.json
```

Run a single test file:
```bash
npx vitest run src/lib/avatarColor.test.js
```

## Architecture

### Content (Starlight docs)

All `.md`/`.mdx` files under `src/content/docs/` become routes. Three sections:

- `guides/` — manually listed in `astro.config.mjs` sidebar
- `reference/` — autogenerated sidebar (`autogenerate: { directory: 'reference' }`)
- `good-to-know/` — autogenerated sidebar

Sidebar config lives in `astro.config.mjs`. To add a new autogenerated section, add an entry there and create the matching directory under `src/content/docs/`.

### Issues Feature (custom pages)

GitHub issues are rendered as threaded dialogue pages outside the Starlight content system:

- `scripts/fetch-issues.mjs` — fetches issues + comments from GitHub API, writes to `src/data/issues.json`
- `src/data/issues.json` — data source (committed; update by running fetch-issues)
- `src/pages/issues/[id].astro` — generates one static page per issue using `StarlightPage` wrapper
- `src/components/IssueThread.astro` — renders a single issue thread; accepts `id` prop
- `src/components/Thread.astro` / `Reply.astro` — presentational components

To embed an issue thread in an MDX doc:
```mdx
import IssueThread from '../../components/IssueThread.astro';

<IssueThread id={42} />
```

### Tests

Vitest covers `src/**/*.test.{js,mjs}` and `scripts/**/*.test.mjs`. Tests import pure functions (e.g. `parseArgs`, `transformIssue` from the fetch script; `avatarColor` from `src/lib/`). No browser environment needed.

## Conventions

- **Base path**: `/BCOEM-Documentation` — always use base-relative links; never hardcode absolute paths to assets
- **Filenames**: all docs filenames must be **lowercase** — a pre-push hook enforces this (Linux CI is case-sensitive)
- **Content collection dir**: `reference/` (singular) — not `references/`
- **MDX**: escape CSS `{` braces as `\{`; do not put raw Markdown lists inside JSX components — use `<ul>/<li>` instead
- **Issue body rendering**: bodies come from GitHub as HTML, rendered via `set:html` — do not re-parse as Markdown

## Deployment

Push to `main` → GitHub Actions builds and deploys to GitHub Pages automatically (`.github/workflows/deploy.yml`). The build runs `npm run build` with `NODE_ENV=production`.
