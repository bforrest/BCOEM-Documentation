# Docs Health System ‚Äî Design Spec

**Date:** 2026-04-12  
**Status:** Approved  
**Repo:** BCOEM-Documentation (Starlight/Astro)

---

## Overview

An autonomous docs health system that audits the Starlight site for common problems on every PR and weekly, automatically fixes what it finds, and opens a PR with the results. Built on a deterministic audit script (no AI) + a `claude -p` fix loop that iterates against `npm run build` until clean or exhausted.

---

## Components

Five files are added to the repo:

```
scripts/
  audit.mjs                          ‚Üê deterministic auditor, emits audit-report.json
  autofix.sh                         ‚Üê claude -p fix loop + gh pr create
.github/workflows/
  audit-pr.yml                       ‚Üê runs on every PR (fast checks)
  audit-weekly.yml                   ‚Üê runs on Monday 9am UTC (deep checks)
docs/
  audit-report-template.md           ‚Üê PR comment/body format
```

## Audit Script (`scripts/audit.mjs`)

Pure Node, no external dependencies beyond what's already in the project. Invoked as:

```bash
node scripts/audit.mjs [--deep]
```

Writes findings to `audit-report.json` in the project root. Always exits 0 ‚Äî the workflows decide what to do with the report; a non-zero exit would short-circuit GitHub Actions steps unnecessarily.

### Checks ‚Äî PR (always)

| Check | What it does |
|---|---|
| **Broken internal links** | Parses `.md`/`.mdx` files for `[text](path)` and `href` references, resolves relative to `src/content/docs/`, flags non-existent targets |
| **Case-sensitivity** | For every resolved link target, verifies on-disk filename case matches exactly |
| **MDX hazards** | Regex scan for unescaped `{` outside code fences/backticks; raw Markdown list syntax (`- item`) inside JSX component tags |
| **Missing sidebar entries** | Cross-references manually-listed slugs in `astro.config.mjs` against files in `guides/`; flags guide files with no sidebar entry (autogenerated sections excluded) |
| **Base-path mistakes** | Flags hardcoded `/BCOEM-Documentation/` in link `href` or image `src` attributes |

### Checks ‚Äî Weekly only (`--deep`)

| Check | What it does |
|---|---|
| **Image path validation** | Finds all image references in `.md`/`.mdx`, verifies targets exist in `src/assets/` or `public/` |
| **Schema drift** | Parses frontmatter via `gray-matter`, validates required Starlight fields (`title` required, `description` optional string) |

### Output shape (`audit-report.json`)

```json
{
  "timestamp": "2026-04-12T09:00:00Z",
  "deep": false,
  "summary": {
    "totalFindings": 3,
    "passed": ["case-sensitivity", "base-path", "sidebar"],
    "failed": ["broken-links", "mdx-hazards"]
  },
  "findings": [
    {
      "check": "broken-links",
      "file": "src/content/docs/guides/example.md",
      "line": 12,
      "text": "[see reference](../reference/Missing.md)",
      "message": "Link target does not exist: src/content/docs/reference/Missing.md"
    }
  ]
}
```

Each finding includes: `check`, `file`, `line`, `text`, `message`.

## Autofix Script (`scripts/autofix.sh`)

```
Usage: scripts/autofix.sh <path-to-audit-report.json>
```

Requires: `claude` CLI in PATH, `gh` CLI authenticated, `ANTHROPIC_API_KEY` in environment.

### Flow

```
1. git checkout -b docs/auto-fix-YYYYMMDD-HHMMSS
2. Build claude -p prompt:
     - Paste audit-report.json findings
     - Paste relevant sections of CLAUDE.md (conventions)
     - Instruction: fix all findings, then run npm run build
3. claude -p \
     --allowedTools "Edit,Read,Glob,Grep,Bash(npm run build)" \
     < prompt.txt
4. Run npm run build
5. If exit 0 ‚Üí BUILD_PASSED=true, break
6. If exit != 0 && attempts < 3 ‚Üí append stderr to prompt, go to step 3
7. If exit != 0 && attempts == 3 ‚Üí BUILD_PASSED=false, continue
8. git add -A
9. git commit -m "docs: auto-fix audit findings [$(date +%Y-%m-%d)]"
10. gh pr create \
      --title "docs: auto-fix $(date +%Y-%m-%d)" \
      --body <filled audit-report-template> \
      [--label "build-failing"]   # only if BUILD_PASSED=false
```

### Claude toolset

Locked to: `Edit`, `Read`, `Glob`, `Grep`, `Bash(npm run build)`. No network access, no arbitrary shell.

### Retry context

Each retry appends the full `npm run build` stderr to the running conversation so Claude has the actual compiler/build error, not a vague failure signal.

### PR label

- Build passing ‚Üí label: `docs-auto-fix`
- Build failing after 3 attempts ‚Üí label: `build-failing` + clear note in PR body: "Build still failing after 3 fix attempts ‚Äî human review needed."

## Workflows

### `audit-pr.yml`

**Trigger:** `pull_request` (types: opened, synchronize, reopened)

**Permissions:**
```yaml
permissions:
  contents: write
  pull-requests: write
```

**Secrets needed:** `ANTHROPIC_API_KEY` (repo secret)

**Steps:**
1. Checkout
2. Node 22 + `npm ci`
3. `node scripts/audit.mjs` ‚Üí `audit-report.json`
4. If `totalFindings > 0`: run `scripts/autofix.sh audit-report.json`
5. Post PR comment using `audit-report-template.md` (pass or fail summary)

**Fix PR targeting:** `main` branch (not the source PR branch ‚Äî avoids fork-push complexity and keeps fixes as standalone reviewable PRs).

**Note:** A PR opened by `GITHUB_TOKEN` will not trigger other workflows (GitHub loop-prevention). This is acceptable ‚Äî the fix PR just needs a human merge.

---

### `audit-weekly.yml`

**Trigger:** `schedule: cron: '0 9 * * 1'` (Monday 9am UTC) + `workflow_dispatch`

**Permissions:** same as above

**Secrets needed:** `ANTHROPIC_API_KEY`

**Steps:**
1. Checkout `main`
2. Node 22 + `npm ci`
3. `node scripts/audit.mjs --deep` ‚Üí `audit-report.json`
4. If `totalFindings > 0`: run `scripts/autofix.sh audit-report.json`
5. If `totalFindings == 0`: exit cleanly with no noise

## Constraints & Assumptions

- `ANTHROPIC_API_KEY` will be added as a GitHub Actions repo secret before the first workflow run
- `gh` CLI is available in the `ubuntu-latest` runner (it is, as of 2024)
- `claude` CLI must be installed in the workflow (`npm install -g @anthropic-ai/claude-code` or equivalent)
- `gray-matter` must be added as a dev dependency (`npm install -D gray-matter`) for the schema drift check
- Fix PRs target `main` ‚Äî if the default branch is renamed, the autofix script needs updating
- The `GITHUB_TOKEN` built-in is used for all GitHub API calls; no PAT required

---

## Out of Scope

- Fixing PRs from forks (GITHUB_TOKEN can't push to fork branches)
- Slack/email notifications (the opened PR is the notification)
- Linting `.astro` component files (TypeScript/ESLint already handles this via `astro check`)
